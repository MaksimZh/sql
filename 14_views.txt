14.1. Чем представление Invoices отличается от таблицы Orders?

Помимо данных из таблиц Orders представление Invoices содержит данные, связанные с каждым заказом, из других таблиц:
 - название службы доставки из таблицы Shippers;
 - название продукта из таблицы Products;
 - имя ответственного сотрудника из таблицы Employees;
 - имя и адрес покупателя из таблицы Customers;
 - стоимость заказа, расчитанная на основе данных из таблицы Order Details.


14.2. Что делает представление "CategorySales for 1997"?

Представление "CategorySales for 1997" суммирует стоимость продаж за 1997 год для каждой категории продукта.
Стоимость по продуктам внутри каждой категории за этот год берётся из представления "Product Sales for 1997".


14.3. Что делает представление "Sales Total by Amount"?

Выбирает заказы на сумму более 2500$, доставленные в 1997 году.
При этом в представлении "Order Subtotals" происходит суммирование по всем товарам в рамках каждого заказа (группировка по [Order Details].OrderID).


14.4. Что делает представление "Products Above Average Price"?

Выбирает продукты, цена которых выше средней.
Средняя цена расчитывается с помощью вложенного SQL-запроса.


14.5. Что делает представление "Summary of Sales by Quarter"?

Выдаёт дату доставки и стоимость для всех заказов, которые уже были доставлены (Orders.ShippedDate не равно NULL).
Вопреки названию никакой группировки по датам здесь не наблюдается.
